---
name: Mobile-first design
description: App is mobile-first — always optimize for small screens, use drawers not dialogs, touch-friendly targets
type: preference
---
- Always use Drawer (bottom sheet) instead of Dialog on mobile viewports
- Touch targets minimum 44px
- Prefer compact, vertically-stacked layouts on mobile
- Test all new UI at 390x844 viewport
- Use `useIsMobile()` hook for responsive component switching
